SchottkyAnoMaLy.DonutVolcanoEnsemble — Type
DonutVolcanoEnsemble{T <: Real}

An ensemble of $M$ donutvolcanos takes the form

\[\mathrm{p}_M(x) = \frac{1}{M} \sum_{k=1}^M \mathrm{donutvolcano}(x, \mu_k, \sigma_k).\]

This object is a wrapper around an ensemble::Vector{Tuple{T, T}} for the individual donutvolcanos. The values of $(μ_k, σ_k)$ are restricted such that $\mu_k \geq 0$ and $\sigma_k > 0$ via the valid function.

SchottkyAnoMaLy.InterpolationSet — Type
InterpolationSet{T <: Real} <: MLMatrixContainer{T}

Container around the relevant arrays used for the interpolations in Kernel Ridge Regression.

Contents

It is assumed that the order of the Chebyshev interpolation is n, the number of ensembles in the interpolation set is N, and the number of temperatures for the specific heat is L.

Runtime constants

These matrices are constants throughout the learning process.

  • specific_heats::Matrix{T}: An L × N matrix containing all the specific heat values for the ensembles in the InterpolationSet.
  • cheby_components::Matrix{T}: an N × (n + 1) matrix of the Chebyshev coefficients for all ensembles in the InterpolationSet.
  • msqdiff_Gram::Matrix{T}: an N × N matrix of the msqdiff values between each of the ensembles in the InterpolationSet.

Runtime containers

These matrices are update!-ed throughout the learning process.

  • gausskernel_Gram::Matrix{T}: an N × N matrix of the gausskernel values between each of the ensembles in the InterpolationSet.
  • deriv_Gram::Matrix{T}: an N × N matrix of the msqdiff values between each of the ensembles in the InterpolationSet element-wise multiplied by those from the full kernel Gram matrix.
  • inv_Gram::Matrix{T}: an N × N matrix used as a temporary placeholder for the intermediary calculations, for example, the inverse of the regularized Gram matrix.

SchottkyAnoMaLy.TrainingSet — Type
TrainingSet{T <: Real} <: MLMatrixContainer

Container around the relevant arrays used for the training of the regularized Kernel Ridge Regression model.

Contents

This structure assumes that there are M ensembles in the TrainingSet and N ensembles in the InterpolationSet. Additionally, it assumes the order of the Chebyshev interpolation is n while the number of temperatures for the training specific heats is L.

Runtime constants

These matrices are constants throughout the learning process.

  • specific_heats::Matrix{T}: An L × M matrix of the specific heat values for each ensemble in the TrainingSet.
  • cheby_components::Matrix{T}: An (n + 1) × M Matrix of true values of the Chebyshev coefficients for all ensembles in the TrainingSet.
  • msqdiff_TvI::Matrix{T}: An N × M matrix containing the msqdiff between each element in the TrainingSet (T) and the InterpolationSet (I).

Runtime containers

These matrices are update!-ed throughout the learning process.

  • gausskernel_TvI::Matrix{T}: An N × M matrix containing the gausskernel between each element in the TrainingSet (T) and the InterpolationSet (I).
  • gausskernel_deriv_TvI::Matrix{T}: An N × M matrix containing the element-wise derivatives of the gausskernel between each element in the TrainingSet (T) and the InterpolationSet (I).
  • predicted_components::Matrix{T}: A container for the (n + 1) × M predicted components from KRR.

SchottkyAnoMaLy.chebydecomposition — Function
chebydecomposition(ensemble, order, Δmin, Δmax, [ coefftol = zero(Float64) ])

Use FastChebInterp.jl to compute the Chebyshev interpolation of a given ensemble to a polynomial order on the [Δmin, Δmax] interval. This function returns the coefficients $a_k$ computed by FastChebInterp.chebinterp and defined by

\[\mathrm{p}(\Delta) \approx \sum_{k=0}^M a_k T_k(\Delta),\]

where $\mathrm{p}(\Delta)$ is the ensemble, $M$ is the order, and $T_k(\Delta)$ is the $k^{\mathrm{th}}$ Chebyshev polynomial of the first kind.

Note

Changing the optional coefftol from the default value of 0.0 may lead to issues downstream. Essentially, FastChebInterp.jl uses this (I think) for precision purposes, but if it is nonzero then the output order is not guaranteed to be what is provided as and argument.

SchottkyAnoMaLy.donutvolcano — Method
donutvolcano(x, μ, σ)

The donutvolcano distribution calculated for an input x with center μ and width σ. The argument x can be a scalar or broadcastable.

The distribution itself is given by

\[{\rm donutvolcano}(x; μ, σ) = \frac{x \Theta(x)}{\mathcal{N}(\mu,\sigma)} \exp\left[ -\frac{\left(x - \mu\right)^2}{2\sigma^2} \right],\]

where we use our own Heaviside Theta function above. The normalization constant $\mathcal{N}(\mu,\sigma)$ given by norm_donutvolcano.

SchottkyAnoMaLy.mass2 — Function
mass2( xdata, ydata, [method = TrapezoidalFast()] )

Use NumericalIntegration.jl to compute the "mean-square-mass" of a set of data ydata over the domain xdata.

The mean-square-mass formula, given as a continuous integral, is

\[m^2[y(x)] = \frac{1}{b - a}\int_a^b \mathrm{d}x\, \left[ y(x) \right]^2.\]

julia> value ≈ 1/(10 - 1) * 1/2 * ( maximum(xvals)^2 - minimum(xvals)^2 )

SchottkyAnoMaLy.minimizing_component — Method
minimizing_component(kernel_column, inv_kernel_matrix, train_component_vector)

Compute the solution to the kernel ridge regression problem (KRR). This solution minimizes the regularized square-error

\[\mathcal{E}_m = \sum_i \left( f^m_i - \sum_j K(i, j; \sigma) \alpha^m_j \right)^2 + \lambda \sum_{ij} \alpha_i K(i, J) \alpha_j,\]

where $K(i, j)$ is the kernel function measured between functions $i$ and $j$, $f^m_i$ is the $m^{\mathrm{th}}$ component of $i^{\mathrm{th}}$ training function, $\sigma$ and $\lambda$ are the hyperparameters, and $\boldsymbol{\alpha}^m$ is related to the $\boldsymbol{f}^m$ training vector as

\[\boldsymbol{\alpha}^m = \left( K(i,j;\sigma) + \lambda \mathbb{1} \right)^{-1} \boldsymbol{f}^m.\]

The prediction that minimizes $\mathcal{E}_m$ is given by

\[f^m_{\mathrm{out}} = \boldsymbol{K}^T_{in} \boldsymbol{\alpha}^m.\]

By direct substitution, this function then computes

\[f^m_{\mathrm{out}} = \boldsymbol{K}^T_{in} \left( K(i,j;\sigma) + \lambda \mathbb{1} \right)^{-1} \boldsymbol{f}^m,\]

where $\boldsymbol{K}^T_{in}$ = kernel_column, the whole inverse is inv_kernel_matrix, and $\boldsymbol{f}^m$ = train_component_vector.

SchottkyAnoMaLy.single_component_loss_gradient — Method
single_component_loss_gradient(prediction, value, νvector, Qmat, Minv, fcomp_vector, ∂σd2_ν_vector)

Compute the loss gradient for a single component (taken analytically of single_component_loss).

This is a hefty function, so it is best to pass in as many of its parts as possible to avoid multiple computations of the same quantity.

The mathematics for this gradient, as a function of the hyperparameters $\sigma$ and $\lambda$ follows as

\[\nabla \ell_m = (f^m - \tilde{f}^m) \begin{pmatrix} \partial_\sigma \tilde{f}^m \\ \partial_\lambda \tilde{f}^m \end{pmatrix}, \]

with

\[\tilde{f}^m = \boldsymbol{\nu}^{\mathrm{T}}(\sigma) \left[ K(\sigma) + \lambda \mathbb{1} \right]^{-1} \boldsymbol{f}^m.\]

$\boldsymbol{\nu}$ is the $N_{\mathrm{interp}} \times 1$ Vector comprised of the gausskernel calculated between the input function and the $N_{\mathrm{interp}}$ functions in the interpolation set. We define the Matrix $M(\sigma, \lambda) \equiv K(\sigma) + \lambda \mathbb{1}$, such that the predicted component $\tilde{f}^m$ simplifies to

\[\tilde{f}^m = \boldsymbol{\nu}^{\mathrm{T}}(\sigma) M^{-1}(\sigma, \lambda) \boldsymbol{f}^m.\]

The quantity $\boldsymbol{f}^m$ is the $N_{\mathrm{interp}} \times 1$ Vector of the $m^{\mathrm{th}}$ Chebyshev component of each ensemble in the interpolation set.

Recalling that

\[\partial (M^{-1}) = - M^{-1} (\partial M) M^{-1},\]

it follows

\[\partial_\lambda \tilde{f}^m = - \boldsymbol{\nu}^{\mathrm{T}} M^{-1} M^{-1} \boldsymbol{f}^m,\]

and

\[\partial_\sigma \tilde{f}^m = -\left[ \sigma^{-3}(d^2\boldsymbol{\nu})^{\mathrm{T}} + \boldsymbol{\nu}^{\mathrm{T}}M^{-1} Q(\sigma, \lambda) \right] M^{-1}\boldsymbol{f}^m.\]

The Vector $\sigma^{-3}(d^2 \boldsymbol{\nu})$ is the element-wise derivative of $\boldsymbol{\nu}$ and the matrix $Q(\sigma, \lambda)$ is that for $K(\sigma, \lambda)$.

SchottkyAnoMaLy.specific_heat — Function
specific_heat([::NLevelSystem = TwoLevelSystem()], T, ::Union{DonutVolcanoEnsemble, FastChebInterp.ChebPoly}, [Δmin = 0, Δmax = Inf])

Calculate the specific_heat $\tilde{c}_V(T, \Delta$ integrated over the DonutVolcanoEnsemble for a fixed temperature $T$. By default, this is calculated for a TwoLevelSystem.

The formula for this integration is given by

\[c_V(T) = \int_{\Delta_{\min}}^{\Delta_{\max}} \mathrm{d}\Delta\, \mathrm{p}(\Delta) \tilde{c}_V(T, \Delta),\]

where $\mathrm{p}(\Delta)$ is calculated from the DonutVolcanoEnsemble.

Note

The FastChebInterp.jl ChebPoly type has bounds for the interpolation. If a Chebyshev interpolation is passed, one must specify (at least) Δmax or else the compiler will complain that the interpolation is outside the ChebPoly bounds.

SchottkyAnoMaLy.update! — Function
update!(gkrr::GaussianKRRML, [hypervals = (σ, λ)])
update!(gkrr::GaussianKRRML, σ, λ) = update!(gkrr, (σ, λ))

Update the GaussianKRRML functor with the Tuple of hypervals. If no parameters are supplied, this function will default to hyperparameters(gkrr) which amounts to a (re)calculation of the dynamic matrices in the InterpolationSet and TrainingSet.

The updates are performed in the following sequence (the order is crucial, thus the need for the functor):

  1. Set the values of gkrr.hyp_σλ (read-only access: hyperparameters(gkrr)).
  2. update! the InterpolationSet (accessed by interpolationset(gkrr)).
  3. update! the TrainingSet (accessed by trainingset(gkrr)).
Note

SchottkyAnoMaLy.update! — Method
update!(intset::InterpolationSet, hypσ, hypλ)

Update the InterpolationSet's dynamic matrices with a new hypσ and hypλ hyperparameters.

The order of the updates is crucial. They follow as

  1. Compute the gausskernel_Gram(intset) with the new hypσ.
  2. Compute the deriv_Gram(intset) with the new hypσ.
  3. From these matrices, compute the inv_Gram(intset) with the new hypλ.
